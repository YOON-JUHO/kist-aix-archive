# KIST AI 아카이브(BBS9157) 리더 / 추천기

AIX 전략실이 운영하는 사내 'AI 아카이브' 게시판 글을 Tampermonkey 수집기가 모아둔
`bbs9157_harvest.json` 에서 읽어, **요약 / 추천 / 본문 읽기**를 해 주는 스킬이다.
2.8MB가 넘을 수 있는 원본을 통째로 컨텍스트에 올리지 말고, 항상 아래 헬퍼 스크립트를
거쳐 필요한 부분만 가져온다.

헬퍼 위치: `scripts/archive.py` (순수 파이썬, 추가 설치 불필요)

## 데이터 소스 (공유 서버 + 자동 최신화)

스크립트는 다음 규칙으로 데이터를 고른다. 보통은 신경 쓸 필요가 없다.
1. `--data <경로>` 또는 환경변수 `AIX_ARCHIVE_JSON` 이 있으면 그 파일을 강제 사용
2. **원내 공유 서버**에서 최신본을 받아 로컬에 캐시한다(`~/.aix-archive-cache.json`). 누군가 수집기로 서버에 올려두면 이게 보통 가장 최신이 된다.
3. 서버가 닿지 않으면 로컬 자동 탐색 — 이전 실행 때 저장된 캐시와
   `~/Downloads` 의 `bbs9157_harvest*.json` 중 **글 최신 날짜가 가장 앞선 파일**로 폴백한다.

**서버 주소는 코드에 들어 있지 않다.** 원내 게시판 공지(Assemble 등)에서 받아 최초 1회 설정한다:
`python archive.py set-url <주소>` (또는 환경변수 `AIX_ARCHIVE_URL`). 설정은 `~/.aix-archive-config.json` 에 저장된다.
사용자가 서버 주소를 모르면, 원내 게시판 'AI 아카이브' 공지를 참고하라고 안내한다.

**첫 실행은 원내망에서 해야 한다**(서버에서 한 번 받아야 캐시가 생긴다).
이후에는 원외에서도 캐시로 동작한다. 데이터가 전혀 없으면 스크립트가 안내 메시지(주소 설정·원내망)를 내니 그대로 사용자에게 전달하면 된다.

따라서 **한 사람이 수집해 서버에 업로드하면, 모든 사용자의 스킬이 같은 최신 데이터로 답한다.**
서버 연동을 끄려면 `AIX_ARCHIVE_URL=""` 로 두면 된다(그럼 로컬 파일만 사용).

**중요 — 답하기 전에 데이터 신선도를 항상 확인한다.** 요약·추천·검색 등 무엇을 하든
먼저 `stats` 를 실행해 어떤 소스가 쓰이는지와 수집 기간(특히 가장 최근 글 날짜)을 확인하고,
답변에 기준 시점을 한 줄로 알려 준다. 예: "기준 데이터: 2026-06-09까지 수집분".
이 스킬은 게시판에 실시간 접속하지 못하므로(수집은 브라우저가 담당), 더 최신 글이 필요하면
재수집이 필요하다고 안내한다.

## 명령어

```bash
python scripts/archive.py stats                       # 글 수, 기간, 작성자 분포
python scripts/archive.py list --recent 10            # 최근 글 목록
python scripts/archive.py list --writer 이제현 --since 2026-05-01
python scripts/archive.py search "질문/키워드" --top 5  # 관련 글 검색(한글+영문 BM25)
python scripts/archive.py show <글id 또는 list/search 순번>   # 본문 전체
```

검색은 형태소 분석기 없이 한글 2-gram + 영문 토큰으로 점수를 매긴다. 결과가 비면
사용자에게 더 일반적인 키워드를 제안하고 한 번 더 검색한다.

## 작업별 절차

### 1) "최근 글 정리해줘" / "한줄평으로 보여줘" (요약·다이제스트)

1. `list --recent N` 으로 목록을 받는다(기본 10건 정도가 적당, 사용자가 기간/작성자
   지정 시 `--since`/`--until`/`--writer` 사용).
2. 각 글이 무슨 내용인지 알아야 한줄평을 쓸 수 있으면, 제목만으로 충분치 않은 글에 한해
   `show <순번>` 으로 본문을 읽는다. 제목이 이미 명확한 공지/안내는 굳이 본문을 열지 않아도 된다.
3. 날짜순으로 **글마다 한 줄 요약**을 만든다. 형식 예:

   ```
   • 06-09 [Oh My ClaudeCode 설치 가이드] (김보금)
     → 여러 에이전트를 병렬로 굴리는 Claude Code 플러그인 OMC 설치법. 토큰 소모 큼 주의.
   ```

   원문 표현을 그대로 베끼지 말고 직접 풀어 쓴다. 핵심 1가지 + (있으면) 주의점/실용 포인트 1가지.
4. 끝에 "더 자세히 볼 글 있으면 제목이나 번호로 알려달라"는 식으로 짧게 마무리한다.

### 2) "이거 관련 글 있어?" / "~~ 어떻게 하는지 아카이브에서 찾아줘" (추천)

1. 사용자의 질문/관심사를 그대로 `search "..."` 에 넣는다. 자연어 질문 전체를 넣어도 된다.
2. 상위 결과 중 **실제로 관련 있는 것만** 골라(점수가 낮거나 스니펫이 무관하면 제외)
   3건 내외를 추천한다.
3. 각 추천 글에 대해: 왜 이 질문에 도움이 되는지 한 문장 + 글에서 얻을 핵심 1~2가지.
   필요하면 `show` 로 본문을 확인한 뒤 정확히 짚어 준다.
4. 추천 글의 작성자·날짜·제목을 함께 표기해, 사용자가 게시판에서 원문을 찾을 수 있게 한다.

### 3) 특정 글 깊이 읽기

`show <id 또는 순번>` 으로 본문 전체를 읽고, 사용자가 원하는 수준(3줄 요약 / 실행 단계 /
핵심 링크 추출 등)에 맞춰 정리한다. 링크(URL)는 본문에 있으면 빠뜨리지 말고 그대로 전달한다.

## 출력 원칙

- 한국어로, 간결하고 실용적으로. 불릿/표는 정보가 많을 때만.
- **원문 문장을 그대로 옮기지 말고** 직접 풀어 쓴다(요약·재서술).
- 추측하지 않는다. 본문에 없는 사실을 지어내지 말고, 데이터에 없으면 없다고 말한다.
- 작성자가 누구인지(특히 이제현 실장/AIX 전략실 공지인지)는 맥락상 유용하면 함께 알려 준다.
- 사용자가 "이런 게 올라오면 알려줘" 같은 상시 모니터링을 원하면, 이 스킬은 마지막으로 받아온
  데이터 기준이며 실시간 알림은 아니라는 점을 정확히 안내한다(새 글 반영은 담당자의 재수집 필요).
